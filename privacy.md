# Privacy Policy for PR Review Guard

Last updated: 2026

## Overview

PR Review Guard is designed to help engineering teams improve code review quality by enforcing a configurable cooldown before pull request approvals.

This extension is built with privacy in mind and does **not collect, store, transmit, or share any personal data.**

---

## Data Collection

PR Review Guard **does not collect any user data**, including but not limited to:

- Personally identifiable information  
- Browsing history  
- User activity  
- Authentication data  
- Financial information  
- Health information  
- Location data  

No analytics, telemetry, tracking scripts, or external monitoring tools are used.

---

## Data Processing

All functionality is executed locally within the user's browser.

The extension only reads the information necessary to detect pull request pages on approved internal Azure DevOps domains in order to enforce the review cooldown feature.

**No data is transmitted externally.**

---

## Storage

The extension uses Chrome's local storage solely to save configuration settings such as the cooldown duration.

These settings:

- remain on the user's device  
- are never sent to any server  
- are not shared with third parties  

---

## Third-Party Access

PR Review Guard does not:

- sell user data  
- transfer user data  
- share data with third parties  

The extension operates entirely client-side.

---

## Remote Code

PR Review Guard does not execute remote code.

All scripts required for the extension are packaged within the extension bundle.

---

## Intended Use

This extension is intended for internal engineering workflows and is limited to approved domains required for pull request review processes.

---

If the privacy policy changes in the future, updates will be reflected on this page.
